# Uni-Controller 安卓 App 使用说明

**版本**：2026.02.02
**适用设备**：基于 的柔性测斜仪控制器
**包名**：`com.rasber.controller`
**连接方式**：蓝牙低功耗（BLE）

---

## 1. 安装与环境要求

### 手机要求
- Android 8.0（API 26）及以上
- 支持蓝牙低功耗（BLE）

### 安装步骤
1. 拷贝安装包 `UniController.apk` 到手机
2. 打开安装，若提示「未知来源」需在系统设置里允许
3. 首次打开 App 会申请以下权限，全部允许：
   - **蓝牙**（扫描 / 连接设备）
   - **定位**（Android 系统要求，BLE 扫描强制依赖位置权限）

### 启动前检查
- 手机蓝牙已打开
- 手机定位已打开（系统级开关，不是 App 级）
- 待连设备已上电，BLE 正在广播（设备名默认 `UniControl`）

---

## 2. 界面总览

App 底部有 4 个标签页，从左到右：

| 标签 | 用途 |
|---|---|
| **连接** | 扫描和连接设备、同步时间、断开 |
| **配置** | 传感器地址批量刷写、型号设置（量产场景） |
| **数据** | 扫地址、读数据、模型管理（调试/巡检） |
| **设置** | 设备参数、网络配置、系统控制 |

**关键规则**：进入「配置/数据/设置」之前，必须先在「连接」页完成蓝牙连接，否则所有操作不生效。

---

## 3. 连接页

### 扫描设备
1. 点击「**扫描**」按钮
2. 扫描窗口 15 秒，列表会实时显示附近 BLE 设备
3. 找到目标设备（设备名 `UniControl` 或自定义名），点击连接

### 连接状态
- 顶部会显示连接指示，绿色表示已连接
- 设备断开时顶部会出现**红色横幅**提示

### 同步时间
连接成功后点「**同步时间**」—— App 会把手机当前时间（含时区）下发给设备 RTC。
**什么时候需要用**：新设备首次部署、设备长期离线后重新上线、模块对时异常时。

### 断开
手动断开用「**断开连接**」按钮；切出 App 一般也会自动断开。

## 4. 配置页（量产场景）

这一页是出厂/批量部署时用的，核心工作流是给成批的传感器分配地址、设型号。

### 工作流 A：广播读（A4）——先摸底
点「**A4 广播读**」→ 选 COM 口（COM1 / COM2）→ 发起读取。
设备会广播读取总线上所有传感器，返回：
- AutoID（出厂唯一 ID）
- A / B / Z 轴校准系数

结果回填到后续步骤的地址输入框，省一次手抄。

### 工作流 B：单点改地址（A6 / A7）
- **A6（一对一改地址）**：指定单个传感器的新地址，同时可设型号
  - 型号下拉含 3 轴 / 2 轴 / 3 轴固定 等 8 种选项
  - 点「**地址 +1**」按钮可自动递增新地址，方便连续刷写
- **A7（重命名地址）**：已有旧地址、改新地址，带回读校验

### 工作流 C：Modbus ID 设置
按地址逐个设 Modbus ID。

### 工作流 D：批量刷地址
适合流水线场景：
1. 设定 AutoID 扫描范围（起/止）
2. 设定起始地址、最大地址
3. 设定设备间延时（防止总线冲突）
4. 点击开始，App 扫 AutoID，匹配一个设备就分配一个递减地址
5. 进度条 0–1024，结果表显示最近 20 条

还可以**从批量中读型号**、**给所有设备统一写型号**。

## 5. 数据页（调试/巡检）

### COM 口切换
顶部切 COM1 / COM2，所有操作只作用于当前选中口。

### 读地址 vs 扫地址
- **读地址**：从设备配置里拉出**已经登记**的传感器列表（秒级）
- **扫地址**：**从 0 到 1024** 逐个探测总线上实际在线的传感器（分钟级，有进度条）

通常先「读地址」看配置，对不上再「扫地址」确认硬件。

### 读数据
- **全部轮询**：点「读数据」，对当前 COM 上所有传感器逐个读 A/B/Z
- **单点轮询**：在表格里点某个地址的那一行，只读这一个

### 数据表
列依次是：地址（可点）· 状态 · AutoID · A(X) · B(Y) · Z · 型号。
状态图标含义：
- `✅` 读成功
- `📍` 已找到但未读
- `❌` 读失败（离线、校验错、地址冲突）
- `🔧` 型号读取成功
- `⏳` 正在处理

### 型号管理
- **全部读型号**：遍历所有传感器取型号
- **全部设型号**：选择型号后一键推到所有传感器

## 6. 设置页

### 设备身份
- **设备 ID**：云端用这个识别设备，可编辑
- **内存剩余**：查看设备运行时的 RAM
- **固件版本**：只读，当前设备固件号

### 系统参数
| 项 | 说明 |
|---|---|
| 间隔预设 | 1=5 分, 2=10 分, 3=15 分, 4=30 分, 5=1 时, 6=2 时, 7=4 时, 8=12 时, 9=24 时, 99=自定义, 0=待机 |
| 自定义间隔 | 预设=99 时生效，单位分钟 |
| 休眠模式 | `light` 浅眠（更快响应） / `deep` 深眠（更省电） |
| RS485 扩展 | 是否启用 SC16IS752 扩展口（COM3/COM4） |
| 合并段 | 上传数据时是否合并多段 |

### 网络配置

#### WiFi
- SSID + 密码
- 启用/禁用开关

#### 4G
- **APN**：运营商接入点
  - 中国移动：`cmnet`
  - 中国联通：`3gnet` 或 `uninet`
  - 中国电信：`ctnet`
  - 海外按运营商填
- **COPS**：运营商选择，`0` = 自动
- **模块型号**：`A7670C`、`A7670G`、`SIM7672E` 等
- **协议**：`yundtu`（云DTU 封装）/ `simcom`（裸模块原生 AT）
  > 不清楚填哪个就先按设备实际硬件：云DTU → `yundtu`；裸 A7670G/SIM 模块 → `simcom`

#### MQTT
- Broker IP、Port
- 发布 Topic
- 用户名、密码
- 启用/禁用开关

### 本地存储
- 启用后，断网时数据会先缓存到设备 flash
- 存储周期：`month`（月）/ `day`（日）粒度清理

### 控制按钮

| 按钮 | 说明 |
|---|---|
| **读取配置** | 从设备拉当前配置到 App 界面 |
| **写入配置** | 把界面上的配置推给设备（设备 NVM 持久化） |
| **导入地址表** | 从手机存储导入 CSV 格式的传感器地址表 |
| **加载默认配置** | 一键恢复出厂默认（慎用） |
| **USB 读写切换** | 切换日常模式 / flash 模式（**重启后生效**） |
| **重启设备** | 软重启 |
| **采集** | 立即触发一轮数据采集和上传（不等定时周期） |

## 7. 常见操作流程速查

### 流程 1：全新设备首次部署
1. 设备上电，打开 App
2. 「连接」→ 扫描 → 连接目标设备
3. 「连接」→ 同步时间
4. 「设置」→ 填设备 ID、网络（WiFi 或 4G）、MQTT → 写入配置
5. 「数据」→ 扫地址 → 确认传感器在线
6. 「设置」→ 采集 → 观察 MQTT 后台是否收到数据

### 流程 2：换了 4G 模块（比如 A7670C → A7670G）
1. 连接设备
2. 「设置」→ 网络 → 4G 段改：模块型号 `A7670G`、协议 `simcom`
3. 「写入配置」
4. 「重启设备」

### 流程 3：出厂批量刷地址
1. 连接设备
2. 「配置」→ A4 广播读 → 摸清总线上哪些传感器在线
3. 「配置」→ 批量刷地址 → 设参数 → 启动
4. 完成后「数据」→ 扫地址验证

### 流程 4：现场排查"上不了云"
1. 连接设备，「设置」→ 读取配置，核对 MQTT 地址/端口/账号
2. 「设置」→ 同步时间（TLS/证书校验依赖时间）
3. 「设置」→ 采集 → 看日志反馈
4. 如果设备离线，用串口（USB CDC）看 `[4G]` / `[Upload]` 日志

## 8. 故障排查

### 扫不到设备
- 蓝牙开了吗？定位开了吗？
- 设备离手机太远或设备没开机
- 设备可能被其他手机连着（BLE 一对一连接）
- 重启 App，重启手机蓝牙

### 连上就断
- 设备正在其他任务中（如批量刷写），稍后重试
- 距离过远导致信号弱

### 写入配置失败
- 检查字段格式（IP、端口要是数字、SSID 别带特殊符号）
- 设备 NVM 空间不足：去掉一些非必要字段再写

### 数据表全红 `❌`
- COM 口选错了
- 传感器未上电或 RS485 接反
- 波特率不匹配（去设置页核对 RS485 波特率）

### 时间显示异常（1933 年之类）
- 固件时间解析遇到模块默认值（2070-01-01）导致 RTC 溢出
- 解决：物理断电重启 → 连接 App → 同步时间

---

## 9. 注意事项

- **蓝牙连接是一对一**：同一时间只有一台手机能连一台设备
- **写入配置后不会立即生效的字段**：USB 读写模式、模块型号切换 —— 需要重启
- **默认配置会清空传感器地址**：点「加载默认配置」前先导出地址表
- **批量刷写过程中不要断开**：中途断开可能导致部分设备地址错乱，需要重新 A4 摸底
